## Cursor overlay

- Prefer planning before implementation.
- If the user asks for a plan, design, or architecture, stay in plan-first behavior until the plan is approved.
- Treat the saved plan and todo list as the execution contract.
- Before implementation, restate the current bounded step, likely files, and validation to run.
- Execute one bounded step at a time. Multiple files are allowed only when they belong to the same coherent step.
- After each meaningful step, compare progress against the plan and report errors, omissions, or discrepancies.
